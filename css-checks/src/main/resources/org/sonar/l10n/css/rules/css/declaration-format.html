For readability reasons, a declaration should comply to the following format:

  • The property and the value should be defined on the same line
  • There should not be any whitespace between the property and the colon
  • There should be one single whitespace between the colon and the value
  • There should be one single whitespace between value and the !important annotation if present
  • There should not be any whitespace between the value (or !important annotation if present) and the ending semicolon

Noncompliant Code Example

h1 {
  color:red;                   /* Noncompliant: missing whitespace between colon and value */
}

h1 {
  color:    red;               /* Noncompliant: too many whitespaces between colon and value */
}

h1 {
  color:
    red;                       /* Noncompliant: property and value not defined on the same line */
}

h1 {
  color: red ;                 /* Noncompliant: unexpected whitespace between value and ending semicolon */
}

h1 {
  color: red !important ;      /* Noncompliant: unexpected whitespace between !important and ending semicolon */
}

h1 {
  color: red!important;        /* Noncompliant: missing whitespace between value and !important */
}

h1 {
  color: red     !important;   /* Noncompliant: too many whitespaces between value and !important */
}

Compliant Solution

h1 {
  color: red;
}

h1 {
  color: red !important;
}
